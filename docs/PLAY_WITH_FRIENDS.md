# MewnBase Multiplayer — как установить и играть с друзьями

Пошаговая инструкция для новичков: установка, игра в одной сети, игра через интернет (Radmin VPN и альтернативы).

> **Важно:** в этом репозитории **нет** файлов самой игры MewnBase (они защищены авторским правом). Нужна **купленная** копия игры (Steam / itch.io) или готовый **portable-сборник**, который вам передал автор мода / ссылка из [Releases](https://github.com/rainbownyashka/mewnbasemultiplayer/releases).

---

## Что понадобится

| Что | Зачем |
|-----|--------|
| Windows 10/11 (64-bit) | Основная платформа сборки |
| MewnBase + патч мультиплеера | Сама игра с сетевым модом |
| Java | Обычно уже внутри portable; иначе Java 8+ |
| Radmin VPN (или аналог) | Игра **через интернет**, если вы не в одной Wi‑Fi/LAN |
| Свободный порт **7777** у хоста | Сервер слушает этот порт по умолчанию |

Рекомендуемый ник: **латиница**, без пробелов (например `Vasya`, `Player1`).

## Вариант A — готовый portable (проще всего)

1. Скачайте архив из раздела **Releases** на GitHub (если там есть `*.zip` с portable — не только `*-src.zip`).
2. Распакуйте в папку без кириллицы в пути, например `C:\Games\MewnBaseMP\`.
3. Запустите **`MewnBase.exe`** или **`run.bat`**.
4. Дождитесь главного меню.

Если игра не стартует — установите [Java 8 или новее](https://adoptium.net/) и снова запустите `run.bat`.

---

## Вариант B — сборка из исходников (для тех, кто клонировал репозиторий)

1. Клонируйте репозиторий:
   ```text
   git clone https://github.com/rainbownyashka/mewnbasemultiplayer.git
   ```
2. Положите **оригинальную** игру в папку `basegame/` (как у вас в Steam):
   - `basegame/game/desktop-1.0.jar`
   - `basegame/data/`, `basegame/MewnBase.exe` и остальные файлы дистрибутива
3. Установите **Python 3** и **JDK** (в проекте обычно Java 8+).
4. Из корня репозитория запустите **`run.bat`** — скрипт скомпилирует изменения из `src/` и встроит их в jar.

Папка `basegame/` в git **не хранится** — её нужно собрать у себя локально.

## Схема: кто есть кто

```text
  ХОСТ (сервер)                         ГОСТЬ (клиент)
  ─────────────                         ───────────────
  Запускает игру                        Запускает игру
  MULTIPLAYER →                         MULTIPLAYER →
  Create Server (7777)                  Connect
  Выбирает сохранение                   IP = адрес хоста
  Играет в мире                         Порт = 7777
  Порт 7777 открыт                      Никнейм
```

- **Локально на одном ПК:** IP = `127.0.0.1`
- **В одной квартире / Wi‑Fi:** IP = локальный адрес хоста (`192.168.x.x`)
- **Через интернет:** IP = адрес из **Radmin VPN** (часто `26.x.x.x`)

## Игра в одной сети (LAN / один компьютер)

### Хост

1. Главное меню → **MULTIPLAYER**.
2. **Create Server (7777)**.
3. Выберите сохранение (или создайте новый мир через обычное меню, затем снова зайдите в хост).
4. Дождитесь загрузки мира — вы в игре, сервер уже работает.

### Гость (второй компьютер в той же Wi‑Fi)

1. Узнайте IP хоста на Windows: `Win + R` → `cmd` → `ipconfig` → строка **IPv4** (например `192.168.0.15`).
2. **MULTIPLAYER** → в поле IP введите этот адрес, порт **7777**, свой ник → **Connect**.
3. Подождите синхронизацию мира (может занять минуту на больших сейвах).

### Два окна на одном ПК (тест)

Второму «игроку» укажите IP **`127.0.0.1`**, порт **7777**. Учтите: два полноценных клиента на одной машине тяжелы для ПК.

## Игра через интернет (Radmin VPN) — основной способ

Игра использует прямое TCP-подключение к хосту. Без VPN обычно **не получится**, если у вас нет «белого» IP и проброса порта. **Radmin VPN** создаёт виртуальную локальную сеть — для игры это самый простой вариант.

### Шаг 1 — установить Radmin VPN всем

1. Скачайте [Radmin VPN](https://www.radmin-vpn.com/) с официального сайта.
2. Установите на **каждом** компьютере.
3. Запустите программу.

### Шаг 2 — одна виртуальная сеть

**Вариант «Сеть» (удобно для постоянной компании):**

1. Один человек: **Сеть → Создать сеть** → придумайте имя и пароль → **ОК**.
2. Остальные: **Сеть → Присоединиться к сети** → то же имя и пароль.

**Вариант «Подключиться» (быстро на один вечер):**

1. Хост: **Подключиться** → показывает свой Radmin ID.
2. Друзья: **Подключиться** → вводят ID хоста → принять приглашение.

### Шаг 3 — узнать IP хоста в Radmin

1. Откройте Radmin VPN.
2. В списке участников найдите **хоста** (тот, кто создаёт сервер в игре).
3. Запомните его **IPv4** в Radmin — обычно вида **`26.12.34.56`**.

Этот адрес вводят гости в поле **Enter IP** в игре (не ваш обычный `192.168...` и не IP с сайта «мой IP»).

### Шаг 4 — хост в игре

Те же шаги, что в LAN: **MULTIPLAYER → Create Server (7777)** → выбор сейва → играете.

### Шаг 5 — гости в игре

**MULTIPLAYER** → IP = **Radmin IP хоста** → порт **7777** → ник → **Connect**.

### Шаг 6 — брандмауэр Windows (если не подключается)

На компьютере **хоста**:

1. При первом запуске разрешите **Java** / **MewnBase** в окне брандмауэра.
2. Или вручную: **Брандмауэр Windows** → **Дополнительные параметры** → **Правила для входящих** → разрешить **TCP порт 7777** для Java/игры.

Проверка у хоста в PowerShell:

```powershell
Test-NetConnection -ComputerName localhost -Port 7777
```

`TcpTestSucceeded : True` — порт слушается.

## Альтернативы Radmin VPN

| Программа | Кратко |
|-----------|--------|
| **Hamachi** | Похожая идея: виртуальная LAN, IP из окна Hamachi |
| **ZeroTier** | Создаёте сеть на zerotier.com, всем ставите клиент, IP из панели ZeroTier |
| **Проброс порта на роутере** | Сложнее: проброс TCP 7777 на IP хоста в локальной сети, гости подключаются к **внешнему** IP |

В игре всегда: **IP виртуальной сети хоста + порт 7777**.

## Быстрое подключение через параметры запуска (опционально)

Можно передать IP сразу при старте (удобно для ярлыка):

```text
-Dmewnbase.mp.host=26.12.34.56
-Dmewnbase.mp.port=7777
-Dmewnbase.mp.nick=MyNick
-Dmewnbase.mp.autoconnect=1
```

Устаревший формат (тоже работает): `-Dmewnbase.connect=26.12.34.56:7777`

Пример для `run_client.bat` в папке `basegame/` (если есть):

```bat
java -Dmewnbase.mp.host=26.12.34.56 -Dmewnbase.mp.port=7777 -Dmewnbase.mp.nick=Player2 -Dmewnbase.mp.autoconnect=1 -jar game\desktop-1.0.jar
```

## Частые проблемы

| Симптом | Что проверить |
|---------|----------------|
| Долго «висит» на загрузке / нет мира | Хост точно в игре после **Create Server**? Порт 7777 не занят другой программой |
| «Не подключается» | IP именно из **Radmin**, не `127.0.0.1` у гостя; хост и гость в **одной** VPN-сети; брандмауэр |
| Игроки не видят друг друга | Разные ники; переподключиться; хост не вышел из мира |
| Вылет при входе | Обновите сборку; удалите папку `saves/multiplayer_received` у клиента и подключитесь снова |
| Планета / смена планеты | В мультиплеере переход на другую планету **отключён** — играйте на одной планете |
| Чат | В игре есть сетевой чат; сообщения приходят без долгой анимации набора |

Логи (если просят помочь с багом): папка `basegame/logs/` или запуск с `-Dmewnbase.logfile=client.log`.

## Настройки сервера (по желанию)

В папке сохранения хоста может появиться файл `server.properties`, например:

` saves/<имя_сейва>/server.properties `

Там можно включать/выключать PvP, телепорты (`allowTp`, `allowTphere`) и др. — см. подсказки в игровой консоли (`help`).

## Краткая шпаргалка на один вечер

1. Всем поставить игру + **Radmin VPN**, войти в одну сеть.
2. **Хост:** MULTIPLAYER → Create Server (7777) → сейв.
3. **Хост** говорит друзьям свой **Radmin IP**.
4. **Друзья:** MULTIPLAYER → этот IP, порт **7777**, Connect.
5. Играете; при выходе хоста сервер останавливается — для следующей сессии хост снова создаёт сервер.

## Список серверов онлайн (зеркало на VPS)

Игра **не** ставится на VPS целиком — на сервере только **зеркало** (список «кто хостит»). Хост по-прежнему на своём ПК + Radmin VPN.

- Подробно: **[SERVER_BROWSER.md](SERVER_BROWSER.md)** (VPS 300 MB хватает, RAM зеркала ~20–50 MB).
- Хост: `-Dmewnbase.mp.registry=http://VPS:8080` и `-Dmewnbase.mp.advertiseHost=<Radmin IP>`.
- Гость: тот же `registry` → в игре **Browse online list**.
